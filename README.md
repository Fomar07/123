# СРС №4: Темная/светлая тема с localStorage

Репозиторий проекта: https://github.com/Fomar07/123

Учебный проект по теме переключения светлой и темной темы. Работа объединяет CSS-переменные, JavaScript-логику и Web Storage, а также показывает, как переносить одно и то же решение на несколько страниц.

## Что реализовано

- переключатель светлой и темной темы через JavaScript
- переопределение CSS-переменных для двух тем
- сохранение выбранной темы в `localStorage`
- восстановление темы при повторной загрузке страницы
- системная тема по умолчанию через `prefers-color-scheme`
- общая логика темы для нескольких страниц проекта
- синхронизация темы между вкладками через событие `storage`
- готовый набор вопросов для интерактивного квеста в Kahoot

## Как запустить

1. Откройте репозиторий и скачайте файлы.
2. Запустите `index.html` в браузере.
3. Переключите тему кнопкой.
4. Обновите страницу и проверьте, что тема сохранилась.
5. Перейдите на `guide.html` и убедитесь, что тема применяется там же.
6. Откройте проект в двух вкладках и смените тему в одной из них.
7. Убедитесь, что тема изменилась и во второй вкладке.

## Как работает проект

### CSS-переменные

В корне документа заданы переменные интерфейса: фон, цвет текста, акцент, тени и цвет поверхности. При смене темы меняется набор этих переменных, поэтому компоненты автоматически подстраиваются под новую тему.

### localStorage

При нажатии на кнопку текущая тема записывается в `localStorage` через `setItem`. При загрузке страницы проект читает значение через `getItem` и применяет сохраненную тему.

### Системная тема

Если пользователь еще не выбрал тему вручную, проект использует результат `window.matchMedia('(prefers-color-scheme: dark)')` и подстраивается под системные настройки устройства.

### Синхронизация вкладок

Если тема меняется в одной вкладке, другие вкладки получают событие `storage` и обновляют интерфейс без перезагрузки страницы.

## localStorage и сервер: в чем разница

### localStorage

- хранит данные только в браузере пользователя
- не требует базы данных и авторизации
- подходит для персональных настроек интерфейса
- не синхронизируется между разными устройствами автоматически

### Серверное хранение

- сохраняет настройки в базе данных или профиле пользователя
- позволяет переносить тему между разными устройствами после входа в аккаунт
- требует серверной логики, API и авторизации
- лучше подходит для больших приложений с учетными записями

# Kahoot-квест для СРС №4

## Название

Разработчик спасает проект: светлая и темная тема

## Описание квеста

Главный герой получает задание внедрить темную и светлую тему в веб-проект. На каждом этапе он должен выбрать правильное решение: как задать CSS-переменные, как сохранить выбор пользователя в `localStorage`, как восстановить тему после перезагрузки и как синхронизировать состояние между вкладками.

## Вопросы для Kahoot

1. Какой способ удобнее всего использовать для смены цветов темы во всем интерфейсе?
Правильный ответ: CSS-переменные.
Неправильные варианты: inline-стили у каждого элемента; случайные классы без общей логики; изменение HTML-тегов.

2. Где лучше хранить выбранную тему пользователя в этом учебном проекте?
Правильный ответ: в `localStorage`.
Неправильные варианты: только в `alert`; в названии файла CSS; в комментарии HTML.

3. Какой метод читает сохраненное значение из `localStorage`?
Правильный ответ: `getItem()`.
Неправильные варианты: `pushItem()`; `readTheme()`; `storageOpen()`.

4. Какой метод записывает выбранную тему в `localStorage`?
Правильный ответ: `setItem()`.
Неправильные варианты: `saveNode()`; `appendTheme()`; `storeValueToHtml()`.

5. Что должно происходить при повторной загрузке страницы?
Правильный ответ: тема должна восстанавливаться из сохраненного значения.
Неправильные варианты: тема всегда должна становиться светлой; тема должна исчезать; кнопка должна удаляться.

6. Что используется в проекте как значение по умолчанию, если тема еще не выбрана?
Правильный ответ: системная тема через `prefers-color-scheme`.
Неправильные варианты: случайный цвет; тема преподавателя; только темная тема.

7. Какой JavaScript-инструмент помогает узнать системную тему устройства?
Правильный ответ: `window.matchMedia()`.
Неправильные варианты: `window.openTheme()`; `document.queryTheme()`; `screen.readMode()`.

8. Что делает событие `storage`?
Правильный ответ: помогает синхронизировать изменения `localStorage` между вкладками.
Неправильные варианты: перезагружает компьютер; меняет HTML на сервере; удаляет CSS.

9. Что лучше для масштабирования на несколько страниц?
Правильный ответ: использовать общий JS-файл и общие CSS-переменные.
Неправильные варианты: копировать разный код на каждую страницу; хранить тему в картинке; менять тему только вручную в DevTools.

10. Что удобнее переключать в теме: значения CSS-переменных или каждый отдельный элемент по одному?
Правильный ответ: значения CSS-переменных.
Неправильные варианты: каждый элемент вручную; только заголовки; только фон страницы.

11. В чем преимущество `localStorage` для темы?
Правильный ответ: выбор сохраняется в браузере между сессиями.
Неправильные варианты: он автоматически публикует сайт; он заменяет сервер; он создает базу данных.

12. В чем ограничение `localStorage` по сравнению с сервером?
Правильный ответ: данные не синхронизируются между устройствами автоматически.
Неправильные варианты: нельзя хранить строки; не работает с CSS; не работает в браузере.

13. Какой вариант лучше описывает серверное хранение темы?
Правильный ответ: настройки можно привязать к аккаунту и получать на разных устройствах.
Неправильные варианты: сервер нужен только для кнопок; сервер хранит только HTML; сервер не связан с настройками пользователя.

14. Какая последовательность логична при запуске страницы?
Правильный ответ: прочитать `localStorage`, если значения нет — взять системную тему, затем применить тему к документу.
Неправильные варианты: сначала удалить стили; сначала показать ошибку; сначала закрыть страницу.

15. Почему в проекте полезно иметь вторую страницу?
Правильный ответ: она показывает, что тема масштабируется и работает во всем проекте, а не только в одном файле.
Неправильные варианты: только чтобы увеличить число файлов; чтобы убрать `localStorage`; чтобы не использовать JavaScript.

## Сюжетная подача для Kahoot

Можно оформить квест как историю из 5 этапов:

- Этап 1: герой выбирает основу для стилизации темы
- Этап 2: герой учится сохранять выбор пользователя
- Этап 3: герой восстанавливает тему при повторной загрузке
- Этап 4: герой добавляет системную тему и синхронизацию вкладок
- Этап 5: герой масштабирует решение на несколько страниц

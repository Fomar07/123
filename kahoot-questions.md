# Kahoot-вопросы

Сюжет: разработчик внедряет темную тему в учебный проект и по шагам настраивает CSS, JavaScript и сохранение темы в браузере.

## Вопрос 1
Тип: выбор правильного варианта

Вопрос: Разработчик хочет вынести основные цвета в одно место. Для чего ему нужны CSS-переменные?

Варианты ответа:
- A. Чтобы хранить пароли пользователя
- B. Чтобы переиспользовать значения цветов и быстро менять тему
- C. Чтобы заменять JavaScript
- D. Чтобы подключать сервер

Правильный ответ: B. Чтобы переиспользовать значения цветов и быстро менять тему

Короткое объяснение: CSS-переменные позволяют задать цвета один раз и затем использовать их в разных частях интерфейса.

## Вопрос 2
Тип: выбор правильного кода

Вопрос: Какой вариант правильно использует CSS-переменную `--bg-color` для фона страницы?

Варианты ответа:
- A. `background-color: --bg-color;`
- B. `background-color: value(--bg-color);`
- C. `background-color: var(--bg-color);`
- D. `background-color: color(--bg-color);`

Правильный ответ: C. `background-color: var(--bg-color);`

Короткое объяснение: Для получения значения CSS-переменной используется функция `var()`.

## Вопрос 3
Тип: поиск ошибки

Вопрос: Разработчик написал код `localStorage.setItem("theme", dark);`. В чем ошибка?

Варианты ответа:
- A. Нужно использовать `getItem()`
- B. Значение `dark` должно быть строкой в кавычках
- C. Нельзя хранить тему в `localStorage`
- D. Нужно писать только в `sessionStorage`

Правильный ответ: B. Значение `dark` должно быть строкой в кавычках

Короткое объяснение: В `setItem()` ключ и значение обычно передаются как строки, поэтому правильно писать `"dark"`.

## Вопрос 4
Тип: выбор правильного варианта

Вопрос: Какой метод получает сохраненную тему из `localStorage`?

Варианты ответа:
- A. `localStorage.pushItem("theme")`
- B. `localStorage.findItem("theme")`
- C. `localStorage.getItem("theme")`
- D. `localStorage.readItem("theme")`

Правильный ответ: C. `localStorage.getItem("theme")`

Короткое объяснение: Метод `getItem()` читает значение по ключу из локального хранилища браузера.

## Вопрос 5

Варианты ответа:
- A. Сначала прочитать тему из `localStorage`, затем применить ее к странице
- B. Сразу удалить все данные из браузера
- C. Подключить базу данных
- D. Перезагрузить страницу второй раз

Правильный ответ: A. Сначала прочитать тему из `localStorage`, затем применить ее к странице

Короткое объяснение: Логика восстановления темы обычно выполняется сразу при загрузке страницы.

## Вопрос 6
Тип: выбор правильного варианта

Вопрос: Пользователь еще не нажимал кнопку смены темы. Как определить предпочтительную тему?

Варианты ответа:
- A. Всегда включать только темную тему
- B. Проверить `prefers-color-scheme`
- C. Случайно выбирать тему
- D. Спросить у сервера без причины

Правильный ответ: B. Проверить `prefers-color-scheme`

Короткое объяснение: Медиа-запрос `prefers-color-scheme` позволяет узнать системную тему устройства.

## Вопрос 7
Тип: выбор правильного кода

Вопрос: Какой код правильно переключает класс `dark-theme` на `body`?

Варианты ответа:
- A. `document.body.classList.toggle("dark-theme");`
- B. `document.body.className.add("dark-theme");`
- C. `document.body.style.toggle("dark-theme");`
- D. `document.body.theme("dark-theme");`

Правильный ответ: A. `document.body.classList.toggle("dark-theme");`

Короткое объяснение: Метод `classList.toggle()` удобно включает и выключает класс по нажатию кнопки.

## Вопрос 8
Тип: последовательность действий

Вопрос: Какой порядок действий правильный, если разработчик хочет сохранить новую тему после клика?

Варианты ответа:
- A. Переключить тему, сохранить новое значение через `setItem()`
- B. Удалить HTML, затем сохранить тему
- C. Сначала закрыть вкладку, потом менять тему
- D. Сначала вызвать `storage`, потом нажать кнопку

Правильный ответ: A. Переключить тему, сохранить новое значение через `setItem()`

Короткое объяснение: После определения новой темы ее нужно применить к странице и записать в `localStorage`.

## Вопрос 9
Тип: поиск ошибки

Вопрос: Разработчик написал `const theme = localStorage.getItem(theme);`. Что здесь неверно?

Варианты ответа:
- A. Ключ `theme` должен быть строкой в кавычках
- B. Нельзя использовать `const`
- C. Метод `getItem()` работает только с числами
- D. Нужно вызывать `setItem()` вместо `getItem()`

Правильный ответ: A. Ключ `theme` должен быть строкой в кавычках

Короткое объяснение: Если имя ключа не записано как строка, JavaScript будет искать переменную `theme`, а не ключ хранилища.

## Вопрос 10
Тип: выбор правильного варианта

Вопрос: Для чего нужно событие `storage` в проекте с темной темой?

Варианты ответа:
- A. Чтобы изменить цвет только в текущем `div`
- B. Чтобы синхронизировать тему между разными вкладками
- C. Чтобы удалить CSS-переменные
- D. Чтобы заменить `localStorage`

Правильный ответ: B. Чтобы синхронизировать тему между разными вкладками

Короткое объяснение: Когда значение в `localStorage` меняется в одной вкладке, другие вкладки могут узнать об этом через `storage`.

## Вопрос 11
Тип: выбор правильного варианта

Вопрос: Какое утверждение правильно описывает отличие `localStorage` от хранения темы на сервере?

Варианты ответа:
- A. `localStorage` хранит тему в браузере пользователя, а сервер хранит ее в удаленной системе
- B. `localStorage` всегда работает быстрее сервера в любых задачах
- C. Сервер не умеет хранить настройки пользователя
- D. Между ними нет никакой разницы

Правильный ответ: A. `localStorage` хранит тему в браузере пользователя, а сервер хранит ее в удаленной системе

Короткое объяснение: `localStorage` подходит для локальных настроек, а серверный способ нужен, когда настройки должны быть доступны на разных устройствах после входа в аккаунт.

## Вопрос 12
Тип: выбор правильного варианта

Варианты ответа:

- C. На каждой странице писать полностью новый код с нуля
- D. Отключить тему на части страниц

Короткое объяснение: Общие стили и единая логика помогают поддерживать проект и не дублировать код.

## Вопрос 13
Тип: выбор правильного кода

Вопрос: Какой вариант правильно определяет системную темную тему?

Варианты ответа:
- A. `window.matchMedia("(prefers-color-scheme: dark)").matches`
- B. `window.colorTheme("dark")`
- C. `document.systemTheme === "dark"`
- D. `navigator.theme("dark")`

Правильный ответ: A. `window.matchMedia("(prefers-color-scheme: dark)").matches`

Короткое объяснение: Через `matchMedia()` можно проверить результат медиазапроса `prefers-color-scheme`.

## Вопрос 14
Тип: последовательность действий

Вопрос: Разработчик открывает сайт впервые. Какой порядок логики самый правильный?

Варианты ответа:
- A. Проверить `localStorage`, если значения нет — взять системную тему, затем применить ее
- B. Сначала включить случайную тему, потом искать сохраненные данные
- C. Сначала очистить `localStorage`, потом применять тему
- D. Ничего не делать до клика пользователя

Правильный ответ: A. Проверить `localStorage`, если значения нет — взять системную тему, затем применить ее

Короткое объяснение: Это самый удобный сценарий: сначала приоритет у выбора пользователя, затем у системной темы.

## Вопрос 15
Тип: поиск ошибки

Вопрос: Разработчик думает, что событие `storage` сработает в той же вкладке, где он вызвал `localStorage.setItem()`. Почему это неверно?

Варианты ответа:
- A. Потому что `storage` обычно отслеживают другие вкладки того же сайта
- B. Потому что `storage` работает только в CSS
- C. Потому что `setItem()` нельзя использовать с темой
- D. Потому что событие работает только после перезагрузки компьютера

Правильный ответ: A. Потому что `storage` обычно отслеживают другие вкладки того же сайта

Короткое объяснение: Вкладка, где тема была изменена, уже знает о своем действии. Событие полезно именно для обновления остальных открытых вкладок.
